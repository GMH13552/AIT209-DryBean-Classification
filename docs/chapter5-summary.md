## 5.1 各维度横向对比

除了精度，我们还可以从多个维度对比这四种算法。

### 精度 vs 泛化

```
                Test Acc    Overfit Gap
SVM             93.31%      +0.95%     ← 高精度 + 低过拟合，理想组合
Random Forest   92.36%      +6.41%     ← 精度不错但过拟合严重
KNN             91.78%      +2.25%     ← 中规中矩
AdaBoost        89.84%      +1.72%     ← 精度最低但泛化尚可
```

SVM 同时做到了最高精度和最低过拟合——这是最理想的状态。

### 推理速度 vs 训练速度

```
                训练时间    推理时间 (单次)
KNN             0.001s      517 ms       ← 零训练，慢推理
SVM             0.7s        610 ms       ← 快训练，慢推理
Random Forest   1.4s         45 ms       ← 中等训练，快推理
AdaBoost        5.6s         30 ms       ← 慢训练，最快推理
```

不同的速度模式对应不同的部署场景：

- **离线批处理**：SVM 或 KNN 都可以
- **在线低延迟**：RF（45ms）或 AdaBoost（30ms）
- **快速原型**：KNN（零训练启动）

### 对噪声的敏感性

虽然没有做系统的鲁棒性实验，但从算法原理可以推断：

- **KNN**：对特征噪声敏感（距离计算），但对标签噪声不敏感（邻居投票有平滑效应）
- **SVM**：RBF 核对异常值有一定鲁棒性，支持向量会自动忽略边缘外样本
- **Random Forest**：Bagging 策略天然抗噪声，部分树被污染不影响整体投票
- **AdaBoost**：对标签噪声最敏感——因为它会放大被分错样本的权重

### 可解释性

- **KNN**：可以展示"最近的邻居是谁"，但多维空间中难以直观理解
- **SVM**：支持向量提供了决策边界的"关键样本"，但 RBF 核的黑箱性降低了可解释性
- **Random Forest**：可以输出特征重要性排名，知道哪些特征最关键
- **AdaBoost**：类似，每轮迭代可以看出哪些样本"最难学"

## 5.2 收获与心得

### 1. 完整的机器学习流程不是调参

这个项目让我真正理解了 ML 工程师的日常工作：**80% 时间花在数据上**。

```
实际耗时分布:
  数据清洗    ████████████████████  60%
  特征理解    ████████              20%
  模型训练    ███                   10%
  文档展示    ███                   10%
```

### 2. 脏数据是常态

教师注入的 4 类数据问题看似简单，但每一个都对应真实的工程场景：

- `?` 占位符 → 不同系统间的数据格式转换
- ` cm` 后缀 → 数据导入导出时的格式错误
- 缺失值 → 传感器故障、人工录入遗漏
- 标签混乱 → 多源数据合并、标注标准不统一

### 3. 没有万能算法

SVM 精度最高但推理慢，RF 推理快但过拟合，AdaBoost 轻量但对噪声敏感。**选择算法就是在精度、速度、鲁棒性之间做权衡。**

### 4. 标准化不是可选项

对于 KNN 和 SVM，不标准化意味着模型完全失效——距离计算会被大数值特征主导。这是很多初学者的常见错误。

### 5. 工程化思维

模块化代码（分离 data_loader、preprocess、train）、命令行接口（无 UI）、GitHub 版本管理——这些工程实践让项目可维护、可复现、可展示。

## 5.3 课程反馈

### 优点

- **实战导向**：脏数据处理 + 多算法对比，模拟了真实的 ML 工作流
- **评分科学**：数据处理 30%、实验 30%、系统 30%——权重分配与实际工作一致
- **自由度大**：课外算法加分鼓励自主探索
- **GitHub 展示**：训练了版本管理和文档能力

### 建议

- 可以加入深度学习（PyTorch）入门实验
- 建议增加**模型可解释性**（SHAP/LIME）作为加分维度
- 可以提供 GPU 环境用于大规模实验
