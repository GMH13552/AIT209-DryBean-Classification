# 6. 课程总结

## 6.1 学习收获

通过本课程 **AIT209 机器学习与项目实践** 的学习和此次期末项目的完成，我在以下方面获得了显著提升：

### 1. 完整的机器学习项目流程

从零到一完成了一个 ML 工程项目：
```
数据加载 → EDA → 数据清洗 → 特征工程 → 模型训练 → 多维度评估 → 文档输出
```
这比单独的算法课更贴近实际工作——现实中 ML 工程师 80% 的时间花在数据上，而非调参。

### 2. 数据质量意识 ⭐

本次项目最大的收获来自**脏数据处理**：

- 真实数据不会像教科书那样干净
- 缺失值、格式错误、标签混乱是常态
- 数据处理占评分 30% 的设计非常合理
- 学会了用 `pd.to_numeric(errors='coerce')` 和字符串清洗等实用技巧

### 3. 多算法对比思维

掌握了从多维度评估模型的方法：

| 维度 | 方法 |
|------|------|
| 精度 | Accuracy, Precision, Recall, F1 |
| 泛化 | 训练-测试精度 Gap |
| 速度 | 推理耗时统计 |
| 鲁棒性 | 加噪声后精度下降 |

### 4. 工程化能力

- 使用 Git/GitHub 进行版本管理和项目展示
- 模块化代码设计（data_loader, preprocess, train 分离）
- 命令行接口（无 UI）
- README 文档撰写

## 6.2 课程评价

### 优点

- **实践性强**：脏数据 + 多算法对比 + 工程架构的考核设计，全面模拟了真实 ML 工作
- **评分明确**：数据分析 5% + 处理 30% + 实验 30% + 系统 30% + 总结 5%，权重分配合理
- **加分机制**：图表、额外维度、GitHub 展示等加分项鼓励做深做精
- **自由度**：课内算法保底 + 课外算法加分，兼顾基础和探索

### 建议

1. 可以增加深度学习框架（PyTorch/TensorFlow）的入门实验
2. 脏数据的问题描述可以更明确（本次是"探索发现"式，体验很好）
3. 可以提供 GPU 资源用于更大规模模型的训练
4. 建议增加模型可解释性（SHAP/LIME）的考核点

## 6.3 技术心得

1. **No Free Lunch 定理**在实践中得到了验证——SVM 精度最高但推理慢，MLP 泛化最好但需要更多调参
2. **标准化是必修课**——不缩放，SVM 和 MLP 根本无法工作
3. **Boosting 天生过拟合**——在训练集 100% 精度不一定是好事
4. **简单模型不简单**——KNN 无需训练且精度仍有 91.78%，在快速原型阶段很有价值
