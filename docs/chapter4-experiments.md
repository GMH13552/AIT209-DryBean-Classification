# 第四章 · 争鸣：多算法实验

完成了数据清洗和预处理后，我们进入机器学习项目最有趣的部分：让不同的算法在同样的数据上"竞技"。

本章实现 **4 种分类算法**，其中 3 种为课堂所学，1 种（AdaBoost）为自行查阅资料实现的课外算法。每种算法都从原理出发，逐步展开到实现和结果分析。

### 为什么选 AdaBoost 作为课外算法？

课堂已经覆盖了 Bagging（Random Forest）和 Boosting 系列（Gradient Boosting），但 AdaBoost 采用了与之不同的思路——不是拟合残差，而是**动态调整样本权重**。它通过给被分错的样本增加权重，迫使后续弱分类器"关注"这些困难样本。

## 4.1 KNN — 朴素的力量

### 原理

K 近邻（K-Nearest Neighbors）是最"懒"的算法——它甚至不需要训练。给定一个新样本，KNN 在特征空间中找出距离最近的 K 个训练样本，由它们投票决定类别。

```
新样本 → 计算与所有训练样本的距离 → 找最近的 K 个 → 多数投票 → 输出类别
```

### 配置

本实验采用 k=5，欧氏距离。k 值太小容易过拟合，太大则边界模糊——5 是一个经验性平衡点。

### 结果

```
Train Acc : 93.86%
Val Acc   : 91.61%
Test Acc  : 91.78%
Overfit   : +2.25%     ← 轻微的过拟合
Speed     : 517 ms      ← 推理较慢（每次都要算全部距离）
```

### 分析

KNN 在没有训练时间的情况下拿到了近 92% 的精度，证明了数据本身的**类别可分性良好**。但它有两个明显缺点：

- **推理慢**：每次预测都要遍历全部 9,527 个训练样本计算距离
- **对特征缩放敏感**：如果没有标准化，Area 特征会完全主导距离计算

> 💡 KNN 是极好的 **baseline 模型**——如果连 KNN 都表现很差，说明数据本身有问题；如果 KNN 已经很好，说明特征质量不错。

## 4.2 SVM — 寻找最优边界

### 原理

支持向量机（Support Vector Machine）的目标是找到一个超平面，使得不同类别的样本之间的**间隔（margin）最大化**。

```
            ● ●
          ●  ↑  ●           ← 支持向量（离边界最近的样本）
    ● ●  ●   │   ●  ●
  ● ● ●     │← margin →    ← 最大间隔
────────────┼────────────   ← 最优分类超平面
  ○ ○ ○     │     ○ ○
    ○ ○  ○  ↓  ○  ○
          ○  ○
```

对于线性不可分的情况，RBF（径向基）核函数将数据映射到高维空间，在高维中寻找线性边界。

### 配置

本实验采用 RBF 核，C=10（适中的正则化强度），`gamma='scale'`（自动适配特征维度）。

### 结果

```
Train Acc : 93.67%
Val Acc   : 92.72%
Test Acc  : 93.31%    ← 🥇 最优！
Overfit   : +0.95%     ← 泛化能力出色
Speed     : 610 ms     ← 推理偏慢
```

### 分析

SVM 以 **93.31%** 的测试精度拔得头筹，而且过拟合仅 0.95%——训练集和验证集几乎一致。

SVM 在这个数据集上表现出色的原因：
- 16 维特征空间不算太大，RBF 核能有效捕捉非线性关系
- 最大间隔原理天然防止过拟合
- 标准化消除了量纲影响

代价是推理较慢——需要存储支持向量并在预测时计算核函数。

## 4.3 Random Forest — 集体的智慧

### 原理

随机森林是 Bagging（Bootstrap Aggregating）的代表：用不同的随机子集训练多棵决策树，让它们**投票**决定最终结果。

```
训练集 → 随机采样子集1 → 决策树1 ─┐
       → 随机采样子集2 → 决策树2 ─┤
       → 随机采样子集3 → 决策树3 ─┼→ 多数投票 → 最终预测
       →     ...      →   ...   ─┤
       → 随机采样子集n → 决策树n ─┘
```

每棵树在训练时还会在每个节点随机选择部分特征，双重随机性确保了森林的多样性。

### 配置

本实验采用 100 棵树，最大深度 12。

### 结果

```
Train Acc : 98.32%    ← 几乎完美记忆训练集
Val Acc   : 91.91%
Test Acc  : 92.36%
Overfit   : +6.41%     ← ⚠️ 过拟合明显
Speed     : 45 ms      ← 推理很快
```

### 分析

随机森林的训练集精度高达 98.32%，但验证集只有 91.91%——**过拟合 6.41%**。100 棵树、深度 12 的配置对 9,527 条数据来说可能有些过强。

但它有两个突出优点：
- **推理快**：45ms，适合部署
- **无需特征缩放**：树模型天然免疫量纲问题

> 🔧 **改进方向**：降低 max_depth 到 8，或者增加 min_samples_split 来抑制过拟合。

## 4.4 AdaBoost — 聚焦难点

### 原理

AdaBoost（Adaptive Boosting）是 Boosting 家族的另一种思路——它不拟合残差，而是**调整样本权重**。

```
第 1 轮: 所有样本等权 → 训练弱分类器1 → 找出分错的样本
                     ↓
第 2 轮: 给分错样本加权重 → 训练弱分类器2 → 再找出分错的
                     ↓
第 3 轮: 继续调整... → 训练弱分类器3 → ...
                     ↓
最终: 加权投票（准确率高的分类器权重更大）
```

每一轮，被前面分类器分错的样本会被赋予更高的权重，迫使后续分类器"盯紧"这些困难样本。

### 配置

本实验采用 50 轮迭代，基学习器为 max_depth=3 的决策树。

### 结果

```
Train Acc : 90.29%
Val Acc   : 88.57%
Test Acc  : 89.84%
Overfit   : +1.72%     ← 泛化尚可
Speed     : 30 ms      ← 推理最快
```

### 分析

AdaBoost 的精度（89.84%）低于其他三种算法。原因在于：

- **对噪声敏感**：如果训练数据中存在错误标注或离群点，AdaBoost 会错误地把它们的权重放大
- **弱学习器太弱**：max_depth=3 的决策树在 16 维特征上可能不足以捕获复杂模式
- **指数损失**：AdaBoost 使用的指数损失函数对误分类惩罚很重，也容易被异常值干扰

尽管如此，AdaBoost 在这组实验中**推理最快**（30ms），且过拟合程度可控。

> 💡 AdaBoost 的失败其实比成功更有教学意义——它展示了**不是所有 Boosting 都适合所有数据**。Random Forest 的 Bagging 策略更适合有噪声的数据，而 AdaBoost 需要更干净的数据和更谨慎的调参。

## 4.5 综合对比与思考

四个算法的表现一览：

```
Accuracy
93.3% ┤                                              ● SVM
      │
92.4% ┤                                       ● RF
      │
91.8% ┤                                ● KNN
      │
89.8% ┤           ● AdaBoost
      │
      └─────────────────────────────────────────────────→
```

### 几个有趣的观察

**① 最慢的推理反而是最快的训练**

KNN 训练时间几乎为零（它干脆不训练），但推理最慢（517ms）。SVM 训练仅 0.7 秒，推理却要 610ms。反过来，RF 和 AdaBoost 训练稍慢但推理飞快。

**② 过拟合不是均匀分布的**

```
SVM:     +0.95%  ← 几乎不过拟合
KNN:     +2.25%
AdaBoost:+1.72%
RF:      +6.41%  ← 严重过拟合
```

同样是树模型，RF 严重过拟合而 AdaBoost 控制得很好。这是因为 AdaBoost 的浅层树（depth=3）天然限制了复杂度，而 RF 的深度 12 树可以完整记住训练数据。

**③ "最好"不等于"最合适"**

SVM 精度最高（93.31%），但如果部署场景要求低延迟（如实时分类流水线），RF（45ms）可能更合适。AdaBoost（30ms）更快但精度不够。工程中要平衡多个维度。

### 多维对比

| 维度 | SVM | RF | KNN | AdaBoost |
|------|:---:|:---:|:---:|:---:|
| 测试精度 | ★★★★★ | ★★★★ | ★★★★ | ★★★ |
| 泛化能力 | ★★★★★ | ★★★ | ★★★★ | ★★★★ |
| 推理速度 | ★★ | ★★★★ | ★★ | ★★★★★ |
| 训练速度 | ★★★★ | ★★★ | ★★★★★ | ★★★ |
| 鲁棒性 | ★★★★ | ★★★★ | ★★★ | ★★ |
| 可解释性 | ★★ | ★★★★ | ★★ | ★★★ |

> 📌 **No Free Lunch**：没有一种算法在所有指标上都最好。SVM 精度最高但推理慢，RF 综合均衡但过拟合需要调参，KNN 简单但慢，AdaBoost 快但对噪声敏感。选择算法就是在这些维度之间做权衡。
